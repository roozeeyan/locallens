# YUSU LAB: Zero to AI Operator

### Полный гайд по настройке профессионального AI-окружения без технических знаний

---

> **Суть гайда:** Большинство людей используют ИИ как поисковик — написал вопрос, получил ответ. Но есть другой уровень: когда ИИ работает как личный ассистент, который помнит всё о тебе и твоих проектах, слышит твой голос, подключён к твоим сервисам и умеет сам писать код, делать ресёрч и сохранять результаты. Этот гайд — пошаговая инструкция, как перейти с первого уровня на второй. Никаких технических знаний не нужно.

## Шаг 1 — Устанавливаем VS Code

**VS Code** — это бесплатный редактор кода от Microsoft. Нам он нужен не для написания кода, а как "дом" для Claude: именно здесь ты будешь общаться с ИИ, давать ему задачи и видеть результаты.

### Как установить:

1. Открой браузер и перейди на сайт **[code.visualstudio.com](https://code.visualstudio.com)**

> 📸 *Скриншот: главная страница VS Code — большая синяя кнопка "Download for Mac"*

2. Нажми большую синяя кнопку **"Download for Mac"**
3. Файл `.zip` скачается в папку Загрузки
4. Открой `.zip` — появится файл приложения `Visual Studio Code`
5. **Перетащи** этот файл в папку `/Applications` (Программы)

> 📸 *Скриншот: перетаскивание VS Code в папку Applications*

6. Открой VS Code из папки Программы
7. Если появится предупреждение безопасности — нажми **"Открыть"**

✅ **Готово!** VS Code установлен. Ты увидишь тёмное окно с приветствием.

## Шаг 2 — Современный терминал

**Терминал** — это "командная строка", место где компьютер выполняет команды. Стандартный терминал Mac выглядит устаревшим. Установим **Warp** — современный терминал с ИИ-возможностями.

### Как установить:

1. В браузере открой Google и напиши: **`warp terminal`**
2. Перейди на первый результат — официальный сайт Warp
3. Нажми кнопку **"Download"**
4. Установи приложение (перетащи в Программы)
5. При первом запуске — войди через Google аккаунт

> 📸 *Скриншот: сайт Warp Terminal — синяя кнопка Download*

> 📸 *Скриншот: интерфейс Warp — красивый тёмный терминал*

> 💡 **Зачем это нужно:** Warp понадобится позже, когда Claude будет выполнять команды за тебя. Он удобнее стандартного терминала.

## Шаг 3 — Плагины для VS Code

Плагины (расширения) — это дополнения, которые добавляют новые возможности в VS Code. Нам нужны несколько ключевых.

### Как открыть магазин плагинов:

В левой панели VS Code найди иконку с 4 квадратами — это **Extensions (Расширения)**. Нажми на неё.

> 📸 *Скриншот: левая панель VS Code — иконка Extensions выделена*

### Плагины для установки:

#### 1. Claude Code (главный плагин!)

1. В поиске Extensions введи: **`Claude`**
2. Найди **"Claude Code"** от Anthropic (официальный)
3. Нажми синюю кнопку **"Install"**

> 📸 *Скриншот: результат поиска "Claude" в Extensions — первый результат "Claude Code" от Anthropic*

#### 2. Markdown Preview Enhanced

1. В поиске Extensions введи: **`Markdown Preview`**
2. Найди **"Markdown Preview Enhanced"** от Yiyi Wang
3. Нажми **"Install"**

> 💡 Markdown — формат для красивого оформления текста. Этот плагин позволяет видеть красивый предпросмотр документов.

#### 3. GitLens (опционально, но полезно)

1. В поиске введи: **`GitLens`**
2. Установи **"GitLens — Git supercharged"**

### После установки плагинов:

**Закрой и снова открой VS Code** — это важно, чтобы плагины активировались.

> ⚠️ **Обязательно:** Cmd+Q для полного закрытия, затем снова открой VS Code из Dock или Программ.

## Шаг 4 — Входим в Claude и создаём папку проекта

### Вход в Claude:

1. В левой панели VS Code появилась новая иконка — **логотип Claude** (антропный ромбик)
2. Нажми на неё
3. Появится панель входа — нажми **"Sign In"**
4. Откроется браузер — войди в свой аккаунт на [claude.ai](https://claude.ai)
5. Разреши доступ — нажми **"Authorize"**
6. Вернись в VS Code — ты вошёл!

> 📸 *Скриншот: панель Claude в VS Code — кнопка Sign In*

> 📸 *Скриншот: после входа — чат с Claude открыт в правой панели*

### Создаём материнскую папку:

Это главная папка, внутри которой будут все твои проекты. Claude будет работать внутри неё.

1. В Finder создай папку в удобном месте (например, на Рабочем столе)
2. Назови её, например: **`Claude Projects`** или **`My AI Workspace`**
3. В VS Code нажми **"Open Folder"** (или File → Open Folder)
4. Выбери созданную папку
5. Появится диалог **"Do you trust the authors?"** — нажми **"Yes, I trust the authors"**

> 📸 *Скриншот: диалог "Do you trust the authors" с кнопкой Yes*

> 💡 **Логика структуры:** Материнская папка — это "источник". Внутри неё будут папки каждого проекта. Claude видит всё, что лежит в открытой папке.

## Шаг 5 — Голосовое управление

Это одна из самых мощных функций: говоришь задачу вслух — Claude её выполняет. Для этого установим приложение **Superwhisper** с моделью распознавания голоса **Parakeet v3**.

### Как установить Superwhisper:

1. В браузере открой Google и напиши: **`Superwhisper app`**
2. Перейди на официальный сайт приложения
3. Скачай и установи (перетащи в Программы)
4. Открой Superwhisper

> 📸 *Скриншот: сайт Superwhisper — кнопка Download*

### Первый запуск и настройка:

1. Superwhisper попросит разрешения — нажми **"Grant Permission"** для микрофона и специальных возможностей
2. Пройди онбординг приложения
3. В настройках выбери модель: найди **Parakeet v3** и нажми установить
4. Дождись загрузки модели (~несколько минут)

> 📸 *Скриншот: настройки Superwhisper — выбор модели Parakeet v3*

> 📸 *Скриншот: иконка Superwhisper в строке меню*

### Как использовать:

- **Зажми клавишу `fn`** (или настроенный хоткей) — начнётся запись
- **Скажи задачу** чётко и развёрнуто
- **Отпусти** — текст автоматически вставится в поле ввода VS Code
- **Нажми Enter** — Claude начнёт работу

> 🎙️ **Пример:** Зажми fn, скажи *"Привет Клод, изучи что есть в этом проекте и расскажи мне об этом"*, отпусти fn, нажми Enter.

## Шаг 6 — Harness Setup (инструкция для Claude)

**Harness Setup** — это специальный файл-инструкция, который объясняет Claude, как работать с твоими файлами и проектами. Это как "должностная инструкция" для ИИ.

### Как установить:

1. Скачай файл Harness Setup по ссылке из Telegram-канала YUSU LAB
2. Открой папку своего проекта в Finder
3. **Перетащи** скачанный файл прямо в корень папки (не в подпапки!)

> 📸 *Скриншот: файл CLAUDE.md лежит в корне папки проекта*

> ⚠️ **Важно:** Файл должен лежать в корне проекта (самая верхняя папка), а не внутри подпапок.

4. Вернись в VS Code
5. Зажми `fn`, скажи:

> *"Изучи harness setup, установи для моего проекта и объясни простыми словами, для чего мне это полезно"*

6. Нажми Enter и дай Claude время разобраться

> 💡 Claude сам разберётся в файле, настроит всё нужное и объяснит тебе на простом языке, что к чему.

## Шаг 7 — Выбираем модель (Claude Opus)

Claude существует в нескольких версиях. **Opus** — самая умная и мощная. Для серьёзных задач лучше использовать именно её.

### Как переключить модель:

1. В чате Claude в VS Code нажми **`/`** (косая черта)
2. Появится меню команд
3. Найди и выбери **"Model"** или просто найди **"Opus"** в списке
4. Выбери **Claude Opus**

> 📸 *Скриншот: меню команд после нажатия `/` — список моделей*

> 💡 **Совет:** Используй Opus для сложных задач (анализ, код, стратегия). Для простых вопросов Sonnet работает быстрее.

### Включаем правильный режим работы:

В настройках Claude Code в VS Code найди опцию **"Ask before edits"** — это режим, в котором Claude будет спрашивать разрешения перед изменением файлов. Рекомендуем включить на старте, пока осваиваешься.

## Шаг 8 — Подключаем сервисы (MCP — "щупальца" Claude)

**MCP (Model Context Protocol)** — это технология, которая позволяет Claude выходить за рамки чата и работать напрямую с твоими приложениями и сервисами.

Представь: Claude может зайти в Notion и создать там страницу, открыть Figma и поправить дизайн, посмотреть твой Google Calendar и найти свободное время, или подключиться к твоему сайту на Tilda.

### Как подключить сервисы:

1. Открой обычное приложение **Claude** (не VS Code, а отдельное десктопное приложение)
2. Зайди в **Настройки** (шестерёнка или меню)
3. Найди раздел **"Connectors"** или **"Integrations"**
4. Нажми **"Customize"**
5. Выбери все сервисы, которыми пользуешься: Google Drive, Notion, Slack, GitHub, и т.д.

> 📸 *Скриншот: раздел Connectors в настройках Claude — список доступных интеграций*

> 📸 *Скриншот: процесс авторизации подключения (например, Google)*

### Примеры того, что теперь можно делать:

```
"Зайди в мой Google Drive, найди последние 5 документов и скажи, 
что там важного"

"Открой Figma и создай там новую рамку 375x812 для мобильного экрана"

"Посмотри мой Notion, найди все задачи со статусом 'в работе' 
и составь список приоритетов"
```

## Шаг 9 — LLM Wiki (память для Claude)

**LLM Wiki** — это система памяти для Claude. Обычно каждый новый чат — чистый лист, Claude не помнит, что было раньше. LLM Wiki решает эту проблему: Claude начинает накапливать знания о тебе, твоих проектах и предпочтениях.

> 🧠 **Аналогия:** Если обычный Claude — это консультант, которого ты видишь первый раз, то Claude с LLM Wiki — это твой личный помощник, который работает с тобой уже несколько месяцев и знает всё.

### Как установить:

1. В Telegram-канале YUSU LAB найди ссылку на **LLM Wiki**
2. Скопируй ссылку
3. Перейди в VS Code, в чат с Claude
4. Зажми `fn`, скажи:

> *"Изучи вот эту ссылку [вставь ссылку], установи и объясни простыми словами, как мне это использовать"*

5. Нажми Enter

Claude сам:
- Изучит документацию по ссылке
- Установит нужные компоненты
- Настроит систему памяти
- Объяснит, как пользоваться

> 📸 *Скриншот: Claude изучает ссылку и показывает процесс установки*

## Шаг 10 — Second Brain (умные заметки для тебя)

**Second Brain** — это персональная система хранения знаний. Если LLM Wiki — это память для *машины*, то Second Brain — это структурированная база знаний для *тебя*. Все твои идеи, заметки, ресёрч, планы — в одном месте, с умной связью между ними.

### Как установить:

1. В Telegram-канале YUSU LAB найди ссылку на **Second Brain**
2. В VS Code зажми `fn` и скажи (или напечатай):

> *"Сформируй всеобъемлющий код для Claude Code, чтобы он установил мне Second Brain для последующего совместного использования с LLM Wiki. И объясни простыми словами, как мне это всё использовать"*

3. Нажми Enter
4. Claude создаст подробный промт-инструкцию
5. **Скопируй** весь текст ответа
6. Вставь его обратно в чат с Claude в VS Code и нажми Enter

> 📸 *Скриншот: процесс — Claude генерирует промт, ты его копируешь и вставляешь обратно*

> 💡 Claude сначала спроектирует систему, потом установит её. Это может занять несколько минут.

## Шаг 11 — Git (сохраняем всё)

**Git** — это система контроля версий. Она сохраняет каждое изменение в твоих файлах, как "точки сохранения" в игре. Даже если что-то сломается — всегда можно вернуться назад.

> 💾 **Зачем это нужно:** Claude будет создавать и изменять файлы. Git гарантирует, что ничего не потеряется и всегда можно откатить изменения.

### Как установить:

В VS Code зажми `fn` и скажи:

> *"Установи мне Git, и когда закончишь всю работу — сохрани все изменения в Git"*

Claude:
- Проверит, установлен ли Git
- Установит при необходимости
- Создаст первое "сохранение" (коммит) всех твоих файлов

> 📸 *Скриншот: Claude выполняет команды установки Git в терминале*

## Шаг 12 — Скиллы (суперспособности Claude)

**Скиллы** — это готовые "программы поведения" для Claude. Каждый скилл учит его делать что-то конкретное особенно хорошо: писать код, делать ресёрч, анализировать данные, работать с дизайном и т.д.

### Как подобрать скиллы под себя:

В VS Code зажми `fn` и скажи:

> *"Сделай ресёрч релевантных мне скиллов, сделай подборку: какие лучше установить, зачем, почему и как мне их использовать"*

Claude:
- Изучит доступные скиллы
- Спросит тебя о твоих задачах и целях
- Предложит персональную подборку
- Установит выбранные

> 📸 *Скриншот: Claude предлагает список скиллов с описаниями*

### После установки — спроси у Claude:

> *"Что ты уже установил и как мне всем этим пользоваться?"*

Claude даст тебе персональный мануал по всему, что было настроено.

## FAQ — Частые вопросы

**Q: Я ничего не понимаю в коде. Это всё равно для меня?**
A: Да! Весь смысл этой настройки в том, что тебе не нужно ничего понимать. Ты говоришь что хочешь — Claude делает. Код — это просто инструмент, как молоток у плотника.

**Q: Нужна ли подписка Claude Pro?**
A: Базовые функции работают бесплатно. Для серьёзного использования (особенно Opus) — рекомендуем Pro ($20/мес). Это окупается за первую неделю.

**Q: Что если что-то пошло не так?**
A: Спроси Claude прямо: *"Что-то пошло не так с [опиши проблему]. Помоги разобраться"*. Он диагностирует и исправит.

**Q: Только для Mac?**
A: Этот гайд написан для Mac. Для Windows большинство шагов аналогичны, но некоторые детали отличаются (например, fn-клавиша работает по-другому).

**Q: Как обновлять всё это?**
A: Спроси Claude: *"Проверь, есть ли обновления для всего, что ты установил, и обнови"*.
